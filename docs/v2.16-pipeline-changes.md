## Overview

These are the specific prompt changes for the n8n scoring workflows that correspond to the Lens v2.16 "Rebuilder Track" update. Each section maps to a pipeline phase.

---

## Phase 1: Enrichment (Brave Search)

**Add to enrichment prompt, after existing company research instructions:**

```
REBUILD SIGNAL DETECTION (for companies with 100-300 employees):

If the company has 100-300 employees, perform an additional search:
"[company name] CCO OR 'VP Customer Success' OR 'Head of Customer' OR 'Chief Customer Officer' hired OR joins OR welcomes"

Look for evidence that CS/CX leadership was hired within the last 12 months.

If found, add to enrichment output:
- rebuild_signal: true
- rebuild_leader: [name and title]
- rebuild_leader_start: [approximate date]
- rebuild_source: [URL of announcement or LinkedIn profile]

If not found after one search attempt, set rebuild_signal: false.
Do not spend more than one search query on this detection. False negatives are expected
and acceptable — warm intros are the primary channel for rebuild opportunities.
```

## Phase 2: Pre-Evaluation Gates

**Modify the headcount gate:**

```javascript
// BEFORE:
// - If employee_count > 350: HARD DISQUALIFY. Score = 0.
// - If employee_count > 200: Apply -5 penalty, continue to scoring.

// AFTER:
// - If employee_count > 350: HARD DISQUALIFY. Score = 0.
// - If employee_count 200-350 AND rebuild_signal = true: No headcount penalty. Continue to scoring with rebuild_signal flag.
// - If employee_count 200-350 AND rebuild_signal = false: Apply -10 penalty, continue to scoring.
// - If employee_count 150-200 AND rebuild_signal = true: No headcount penalty. Apply +20 rebuild bonus in Phase 5.
// - If employee_count 150-200 AND rebuild_signal = false: Apply -5 penalty, continue to scoring.
```

**Confirm Series C handling:**

```
Series C is NOT a hard gate. It is a -5 penalty applied in Phase 5 scoring.
Do not auto-disqualify Series C companies.
Only Series D, Series E, and Growth stage are hard gates.
```

## Phase 4: CS Hire Readiness

**Replace existing CS Hire Readiness prompt logic:**

```
CS Hire Readiness measures whether the company has a CS LEADERSHIP GAP,
not whether a CS team exists.

Score CS Hire Readiness as follows:

HIGH (20-25 points):
- No CS team exists. Founder handles support. Classic builder signal.
- Frontline CSMs exist but no Director+ CS leader. Leadership gap.
- CS leadership recently turned over (<12 months). New executive is hiring
  their team. This is a REBUILDER scenario.
- Job posting explicitly says "first hire," "build from scratch," "define
  the playbook."

MEDIUM (10-19 points):
- Small CS team exists (2-5 people) with a team lead but no senior leader.
- Company recently posted multiple CS roles simultaneously (scaling signal).
- CS function exists but reporting structure unclear or in flux.

LOW (0-9 points):
- Stable CS leadership in place >12 months with fully staffed team.
- Multiple layers of CS management already exist (Director + Managers).
- JD language is "maintain," "optimize," "inherit," "established playbooks."

CRITICAL: The existence of individual contributor CSMs is NOT automatically
a negative signal. A company with 5 CSMs and no CS Director has HIGHER
readiness than a company with 0 CSMs where the founder answers tickets.
The question is whether there is a leadership gap this candidate would fill.
```

## Phase 5: Full Evaluation + Scoring

**Add Rebuilder to Role Type classification:**

```
ROLE TYPE CLASSIFICATION (three categories):

BUILDER: First hire or greenfield CS/support leadership. No existing team
or team of <3. Language: "build from scratch," "first hire," "define the
playbook," "greenfield," "0 to 1."
Score: Up to 25-30 points in Role Mandate.

REBUILDER: CS function exists but is being redefined under new leadership.
New CCO/VP CS/Head of CS in role <12 months. Role reports to or works
closely with that new leader. Function is being restructured, not maintained.
Language: "transformation," "redefining," "evolving," "building the next
chapter," "standing up a new motion." OR: rebuild_signal = true from
enrichment.
Score: Up to 20 points in Role Mandate.

MAINTAINER: Established CS org with stable leadership. Role is to manage,
optimize, or scale existing operations. Language: "book of business,"
"renewal targets," "existing portfolio," "NRR," "optimize and grow."
Score: Up to 5 points in Role Mandate.

When classifying, look for CONTEXT not just keywords. A JD that says
"manage a team of 5 CSMs" is MAINTAINER language — unless the enrichment
data shows the CCO/VP CS joined 3 months ago, in which case it's
REBUILDER. Context overrides keywords.
```

**Add rebuild bonus to scoring:**

```
REBUILD SIGNAL BONUS:
If rebuild_signal = true (from enrichment): +20 points
This bonus applies in addition to Role Mandate scoring.
It reflects that a new CS executive actively building their leadership
team creates a build mandate even at an established company.
```
